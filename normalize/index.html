This is the coding challenge from Redfin. In the input box underneath, submit a path string, and the table below will display the normalized path. The code for the normalize method is available on my github account.

I know that in the challenge description, it was said that "foo//bar" should normalize to "foo//bar". However, I used regex to split the string between one or more slashes. Then at the end, I join each directory with a slash. It seemed like I would have to work backwards to make "foo//bar" normalize to "foo//bar". My function, instead, normalizes "foo//bar" to "foo/bar". I hope that, Redfin, a company that I believe is adaptive and understanding, would be okay with this alteration.

It was very tempting not to overwork this coding challenge and go through all the possible normalization requirements. Here is a list of assumptions I made:

  • The directory names are valid as they are inputted.
  • All path inputs are relative, instead of absolute.
  • The path "foo/bar" is the same as "foo/bar/", so it's not necessary to strip the "/" at the end.
